Boot Loader 帮助

欢迎使用 SUSE LINUX 9.2

使用此菜单来选择所需的功能。如果您在此帮助系统中导航时出现问题，请按 F1 键进入帮助系统的说明。此菜单中的主要功能有：

从硬盘引导：此选择不会对系统执行任何操作。它只启动以前安装的操作系统。

安装：此安装方式适用于大多数计算机。如果在引导过程中出现系统冻结，或在检测硬件组件（如磁盘控制器或网卡）时出现问题，请尝试以下安装选项。

安装 － 禁用 ACPI：目前市面上的许多计算机的 ACPI 实施都不完整或有问题。此选择在内核中禁用 ACPI 支持，但仍启用许多性能功能，如 IDE 硬盘的 DMA。

安装 - 安全设置：如果安装不成功, 此选择也许能够解决问题。

手工安装：此项主要供高级用户使用。使用此选项可以最大限度地控制安装。例如，在启动应急系统之前装载额外的模块。

应急系统：此引导图象可在 RAM 中启动一个小的 Linux 系统。这在系统没有正常启动时非常有用。引导此系统后，以根用户的身份登录。

内存测试：内存测试不仅用于检查新内存模块的安装，还可用于对计算机系统的许多部分进行压力测试，并可以指出一些硬件问题。

引导选项：引导选项可能完全改变您的系统的行为。它们是针对内核的设置。

F1 帮助：这是上下文相关的。它将根据引导屏幕的活动元素显示不同的屏幕。此外还提供帮助系统的说明。

F2 视频方式：在安装时可使用此功能选择不同的屏幕分辨率。如果图形安装出现问题，则文本方式或许能够解决问题。

F3 安装源：选择安装源。

F4 语言：设置 Boot Loader 使用的语言和键盘映射。

F5 内核启动：正常情况下将看不到任何内核讯息。可使用 F5 键进行更改。

F6 驱动程序更新：对于型号很新的计算机，可能需要驱动程序更新来安装系统。

从硬盘引导

选择引导已安装 OS 来启动在您的本地硬盘上安装的系统。必须正确地安装此系统，因为仅启动第一个硬盘上的 MBR（主引导记录）。第一个硬盘的设备 ID 由计算机的 BIOS 提供。

如果忘记从驱动器中取出 CD 或 DVD 而又希望从硬盘启动计算机，可使用此选项。

安装 － 禁用 ACPI

在 SuSE Linux 中，稳定的 2.4 内核可以得到开发人员内核 2.5 的 ACPI 支持。非常新的硬件有时要求 ACPI 来控制中断处理。ACPI 完全取代了以前的 APM 系统。

如果在内核引导期间出现问题，则可选择安装 － 禁用 ACPI。ACPI 存在问题的计算机的已知问题包括：

* 引导时内核冻结
* 未检测到或未正确初始化 PCI 卡

安装 － 安全设置

如果在安装时出现挂起或不能再现的错误，可选择安装 － 安全设置。此选项禁用 IDE 驱动器的 DMA 和所有电源管理功能。参见 apm、acpi 和 ide 内核选项。

安装程序选项：install

默认情况下，将搜索本地 CD-ROM 中的安装源。对于网络安装，请选择 install 选项。可能的安装协议有
* FTP
* NFS
* HTTP
使用的语法与标准 URL 一样。例如，如果发现您的服务器位于 192.168.0.1，而希望从此服务器上的目录 /install 来进行基于 NFS 的安装，请指定以下安装源：

install=nfs://192.168.0.1/install

将用 dhcp 对网卡进行配置，或者您必须自己指定各参数，如网络选项中所述。


返回到引导选项

内核选项：splash

启动屏幕是在系统启动期间显示的图片。

splash=0

关闭启动屏幕。这对于型号非常老的显示器或在出现某些错误时很有用。

splash=verbose

激活启动屏幕，仍显示内核和引导讯息。

splash=silent

激活启动屏幕，但不显示讯息。而是显示一个进度条。

返回到引导选项

安装程序选项：网络选项

可以立即配置网络接口。YaST2 稍后将检测硬件。配置网络至少需要主机 IP 和网络掩码等选项。例如：

hostip=192.168.0.10 netmask=255.255.255.0

或采用简写形式：

hostip=192.168.0.10/24

如果指定了基于网络安装，但未指定这两个选项，则安装程序会尝试用 dhcp 来配置网络接口。如果需要一个默认网关，请用 gateway 选项来指定。例如：

gateway=192.168.0.8


返回到引导选项

安装程序选项：vnc

要启用 VNC 安装，请指定参数 vnc 和 vncpassword：

vnc=1 vncpassword=example

将启动 VNC 服务器，您可以从远程系统通过任何 VNC 客户程序来控制 YaST2。


返回到引导选项。

内核选项：apm

APM 是当前计算机上使用的两种电源管理策略之一。它主要用于便携式计算机，以实现硬盘休眠等功能，但它也可以负责在关闭电源后关闭计算机。APM 依赖于正常工作的 BIOS。如果 BIOS 出现故障，则 APM 的用途可能有限，甚至会妨碍计算机工作。因此，可以用以下参数将其关闭：

apm=off － 完全关闭 APM

一些型号非常新的计算机可以更多地受益于较新的 ACPI。

返回到引导选项

内核选项：acpi

ACPI （高级配置和电源接口）是一个用于定义操作系统和 BIOS 之间电源和配置管理接口的标准。默认情况下，如果检测到 BIOS 是 2000 年以后的版本，就会打开 acpi。几个常用的参数可以用来控制 ACPI 的行为：

pci=noacpi － 不使用 ACPI 来路由 PCI 中断 acpi=oldboot － 只有与引导相关的 ACPI 部分保持激活状态
acpi=off － 完全关闭 ACPI
acpi=force － 即使您的 BIOS 版本是 2000 年之前的版本，也打开 ACPI

在新计算机上，它将取代以前的 apm 系统。

返回到引导选项

内核选项：ide

与 SCSI 不同，IDE 通常用在大多数台式机工作站中。为了避免 IDE 系统发生的一些硬件问题，请使用以下内核参数：

ide=nodma － 关闭 IDE 驱动器的 DMA


返回到引导选项。
